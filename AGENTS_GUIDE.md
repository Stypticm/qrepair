# 🤖 Руководство по использованию ИИ агентов

## Что такое ИИ агенты в вашем приложении?

ИИ агенты - это автоматические системы, которые:
- **Тестируют** ваше приложение 24/7
- **Анализируют** поведение пользователей
- **Предлагают** улучшения UX
- **Мониторят** производительность
- **Отслеживают** ошибки

## 🧪 Тестировщик Агент

### Что делает:
- Автоматически тестирует все пользовательские сценарии
- Проверяет валидацию форм
- Тестирует расчет цен
- Проверяет обработку ошибок

### Как использовать:

#### 1. Запуск тестов через админ-панель:
```
http://localhost:3000/admin/agents
```

#### 2. Запуск тестов через API:
```bash
# Запустить все тесты
curl -X POST http://localhost:3000/api/agents/testing/run

# Запустить конкретный тест
curl -X POST http://localhost:3000/api/agents/testing/run \
  -H "Content-Type: application/json" \
  -d '{"testName": "complete_happy_path"}'
```

#### 3. Получение статистики:
```bash
curl http://localhost:3000/api/agents/testing/stats
```

### Доступные тесты:
- `complete_happy_path` - Полный путь пользователя
- `device_info_validation` - Валидация IMEI и S/N
- `price_calculation` - Расчет цены
- `form_validation` - Валидация форм
- `error_handling` - Обработка ошибок

## 📊 UX Аналитик Агент

### Что делает:
- Отслеживает поведение пользователей
- Находит точки выхода (где пользователи уходят)
- Анализирует время на страницах
- Предлагает улучшения UX

### Как использовать:

#### 1. Интеграция в компоненты:
```tsx
import { useUXAnalytics } from '@/hooks/useUXAnalytics';

function MyComponent() {
  const { trackPageView, trackButtonClick, trackError } = useUXAnalytics();

  useEffect(() => {
    trackPageView('/request/device-info');
  }, []);

  const handleButtonClick = () => {
    trackButtonClick('start_form', '/request/device-info');
  };

  const handleError = (error) => {
    trackError(error.message, '/request/device-info');
  };
}
```

#### 2. Просмотр статистики:
```
http://localhost:3000/admin/agents
```

#### 3. API для получения данных:
```bash
# Статистика UX
curl http://localhost:3000/api/agents/ux/stats

# Рекомендации
curl http://localhost:3000/api/agents/recommendations
```

## 🎛️ Админ-панель

### Доступ:
```
http://localhost:3000/admin/agents
```

### Что показывает:
- **Статистика тестов**: количество, успешность, время выполнения
- **UX метрики**: сессии, завершенность, точки выхода
- **Рекомендации**: предложения по улучшению
- **Управление**: запуск тестов, обновление данных

### Функции:
- ✅ Запуск всех тестов одной кнопкой
- 📊 Просмотр статистики в реальном времени
- 💡 Просмотр рекомендаций по улучшению
- 🔄 Обновление данных

## 🗄️ База данных

### Новые таблицы:
- `AgentTestResult` - Результаты тестов
- `AgentAnalytics` - UX метрики
- `AgentRecommendation` - Рекомендации

### Просмотр данных:
```sql
-- Последние тесты
SELECT * FROM "AgentTestResult" ORDER BY "createdAt" DESC LIMIT 10;

-- UX метрики
SELECT * FROM "AgentAnalytics" WHERE "agentType" = 'ux_analytics';

-- Рекомендации
SELECT * FROM "AgentRecommendation" ORDER BY "priority" DESC;
```

## 🆘 Решение проблем

### Тесты не запускаются:
1. Проверьте подключение к БД
2. Убедитесь что Prisma Client обновлен
3. Проверьте логи в консоли

### UX данные не сохраняются:
1. Проверьте что хук useUXAnalytics подключен
2. Убедитесь что API endpoints работают
3. Проверьте права доступа к БД

### Админ-панель не загружается:
1. Проверьте что все API endpoints доступны
2. Убедитесь что БД содержит данные
3. Проверьте консоль браузера на ошибки

## 📞 Поддержка

Если у вас возникли проблемы:
1. Проверьте логи в консоли браузера
2. Проверьте логи сервера
3. Убедитесь что все зависимости установлены
4. Проверьте подключение к БД

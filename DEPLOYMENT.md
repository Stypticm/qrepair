# 🚀 Деплой на Vercel

## 📋 Что нужно для деплоя

### 1. **Next.js приложение** ✅
- Уже готово для деплоя на Vercel
- Все API routes настроены
- База данных PostgreSQL подключена

### 2. **Python парсер** ⚠️
- **Локально**: Работает как отдельный сервис на порту 8001
- **На Vercel**: Используется встроенный API route `/api/parse-prices`

## 🔧 Настройка для Vercel

### Переменные окружения в Vercel:
```bash
# База данных
DATABASE_URL=postgresql://...

# Базовый URL (ваш домен Vercel)
NEXT_PUBLIC_BASE_URL=https://your-app.vercel.app

# Python парсер (только для локальной разработки)
PYTHON_PARSER_URL=http://localhost:8001
```

### 1. **Деплой Next.js приложения**
```bash
# Установка Vercel CLI
npm i -g vercel

# Деплой
vercel

# Или через GitHub (рекомендуется)
# 1. Подключите репозиторий к Vercel
# 2. Настройте переменные окружения
# 3. Деплой автоматический при push
```

### 2. **Настройка базы данных**
- Используйте **Supabase** или **PlanetScale** для PostgreSQL
- Добавьте `DATABASE_URL` в переменные окружения Vercel

## 🎯 Как работает парсинг на Vercel

### **Локально:**
1. Запускаете Python парсер: `python main_simple.py`
2. Next.js обращается к `http://localhost:8001`
3. Python парсер делает реальный парсинг сайтов

### **На Vercel:**
1. Next.js использует встроенный API route `/api/parse-prices`
2. Возвращает моковые данные (реалистичные, но не реальные)
3. **Нет внешних зависимостей** - всё работает в Vercel

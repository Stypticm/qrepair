# Инструкции по деплою и обновлению версии

## 🚀 Обновление версии приложения

После каждого деплоя необходимо обновить версию приложения для обхода кэширования Telegram:

### 1. Обновить переменную окружения
В файле `.env.local` или в настройках Vercel обновить:
```bash
NEXT_PUBLIC_APP_VERSION=1.0.1  # Базовая версия
```

**Примечание:** Система автоматически увеличивает последнюю цифру версии на 1 при каждом деплое:
- 1.0.1 → 1.0.2
- 1.0.2 → 1.0.3
- И так далее...

### 2. Обновить inline-кнопки в Telegram
После деплоя бот автоматически будет отправлять новые URL с версией и timestamp.

### 3. Принудительное обновление кэша
Если пользователи все еще видят старую версию:
- Попросить их нажать `/start` для получения новых кнопок
- Или закрыть и снова открыть Web App

## 🔧 Настройки Vercel

### Переменные окружения
Добавить в Vercel Dashboard:
- `NEXT_PUBLIC_WEBAPP_URL` - URL вашего приложения
- `NEXT_PUBLIC_APP_VERSION` - версия приложения

### Очистка кэша
После деплоя в Vercel Dashboard:
1. Settings → Data Cache
2. Нажать "Purge Everything"

## 📱 Telegram Web App

### Кэширование
- Telegram кэширует Web App по URL
- Добавление версии и timestamp обходит кэширование
- Каждый деплой генерирует уникальный URL

### Обновление кнопок
- Inline-кнопки автоматически обновляются
- URL содержит версию и timestamp
- Пользователи получают актуальные ссылки

### Full Screen режим
- **Inline-кнопки** теперь открываются в full screen (`is_visible: true`)
- **Автоматическое разворачивание** при загрузке страницы
- **Настройка цветов** заголовка и фона для лучшего UX
- **Кнопка закрытия** с подтверждением

## 🚨 Важно!

- **Всегда обновлять версию** при деплое
- **Проверять переменные окружения** в Vercel
- **Очищать кэш** после деплоя
- **Тестировать** на разных устройствах

# 🛡️ Предотвращение дублирования данных

## ❌ Проблема
При многократном парсинге одних и тех же устройств в базе данных могли накапливаться дублирующиеся записи в таблице `MarketPrice`.

## ✅ Решение

### 1. **Автоматическая очистка старых записей**
```typescript
// Удаляем записи старше 1 часа перед новым парсингом
const oneHourAgo = new Date(Date.now() - 60 * 60 * 1000)
await prisma.marketPrice.deleteMany({
  where: {
    deviceId: device.id,
    createdAt: { lt: oneHourAgo }
  }
})
```

### 2. **Проверка существующих записей**
```typescript
// Проверяем, есть ли уже такая запись
const existingPrice = await prisma.marketPrice.findFirst({
  where: {
    deviceId: device.id,
    source: priceData.source,
    price: priceData.price,
    title: priceData.title || ''
  }
})

if (existingPrice) {
  // Обновляем существующую запись
  await prisma.marketPrice.update({...})
} else {
  // Создаем новую запись
  await prisma.marketPrice.create({...})
}
```

### 3. **API для ручной очистки**
- **Endpoint**: `POST /api/admin/cleanup-duplicates`
- **Функция**: Удаляет дубликаты и старые записи
- **Доступ**: Только для админов

### 4. **Автоматическая очистка по расписанию**
- Записи старше 7 дней удаляются автоматически
- Дубликаты за последние 24 часа удаляются при ручной очистке

## 🔧 Как использовать

### **Автоматически:**
1. При каждом парсинге старые записи (1+ час) удаляются
2. Новые записи проверяются на дублирование
3. Если запись существует - обновляется, иначе создается новая

### **Вручную:**
1. Откройте админ панель
2. Нажмите "Очистить дубликаты"
3. Система покажет количество удаленных записей

### **Программно:**
```bash
# Очистка дубликатов
curl -X POST https://your-app.vercel.app/api/admin/cleanup-duplicates

# Статистика дубликатов
curl https://your-app.vercel.app/api/admin/cleanup-duplicates
```

## 📊 Мониторинг

### **Проверка дубликатов:**
```sql
SELECT 
  "deviceId", 
  "source", 
  COUNT(*) as count
FROM "MarketPrice" 
WHERE "createdAt" > NOW() - INTERVAL '24 hours'
GROUP BY "deviceId", "source"
HAVING COUNT(*) > 1
ORDER BY COUNT(*) DESC;
```

### **Статистика записей:**
- **Всего записей**: `SELECT COUNT(*) FROM "MarketPrice"`
- **За последние 24 часа**: `WHERE "createdAt" > NOW() - INTERVAL '24 hours'`
- **Дубликаты**: Записи с одинаковыми `deviceId`, `source`, `price`, `title`
